# PR 내용 작성 스킬

현재 브랜치의 변경 사항을 분석하여 PR 설명을 마크다운 형식으로 작성합니다.

## 실행 지침

### 1단계: 브랜치 정보 및 커밋 내역 확인

다음 명령어를 실행하여 변경 사항을 파악하세요:

```bash
# 현재 브랜치명 확인
git branch --show-current

# 메인 브랜치 대비 커밋 내역 확인
git log main..HEAD --oneline

# 메인 브랜치 대비 전체 변경 사항 확인
git diff main..HEAD

# 변경된 파일 목록
git diff main..HEAD --name-status
```

### 2단계: 변경 사항 분류

커밋과 변경된 코드를 분석하여 다음과 같이 분류하세요:

| 분류 | 포함 내용 |
|------|----------|
| **주요 변경사항** | 새 기능, 핵심 로직 변경, 아키텍처 변경 |
| **기타 변경사항** | 버그 수정, 스타일 변경, 리팩토링, 설정 변경 |

### 3단계: PR 내용 작성

아래 템플릿에 맞춰 PR 내용을 작성하세요:

**기본 템플릿 (기타 변경사항이 있는 경우)**:

```markdown
## 주요 변경사항

> 핵심 작업 내용 요약
>
- [변경 내용 1]
- [변경 내용 2]
- [변경 내용 3]

## 기타 변경사항

> 주요 기능 외 수정된 부분 (버그, 스타일 등)
>
- [기타 변경 1]
- [기타 변경 2]
```

**기타 변경사항이 없는 경우 (섹션 생략)**:

```markdown
## 주요 변경사항

> 핵심 작업 내용 요약
>
- [변경 내용 1]
- [변경 내용 2]
- [변경 내용 3]
```

### 작성 규칙

1. **한글로 작성**
2. **명확하고 간결하게**: 각 항목은 한 문장으로
3. **동사형으로 시작**: "추가", "수정", "개선", "제거" 등
4. **구체적으로**: 무엇이 어떻게 변경되었는지 명시
5. **관련 파일/컴포넌트 언급**: 필요시 파일명이나 모듈명 포함
6. **기타 변경사항이 없으면 해당 섹션을 출력하지 않음**

### 예시

**예시 1: 기타 변경사항이 있는 경우**

브랜치: `feature/user-authentication`

커밋 내역:
- feat(auth): 로그인 API 구현
- feat(auth): JWT 토큰 발급 로직 추가
- fix(ui): 로그인 버튼 스타일 수정
- chore: 인증 관련 의존성 추가

PR 내용:

```markdown
## 주요 변경사항

> 핵심 작업 내용 요약
>
- 사용자 로그인 API 엔드포인트 구현 (`/api/auth/login`)
- JWT 토큰 발급 및 검증 로직 추가
- 로그인 폼 컴포넌트 및 상태 관리 구현

## 기타 변경사항

> 주요 기능 외 수정된 부분 (버그, 스타일 등)
>
- 로그인 버튼 hover 스타일 수정
- jsonwebtoken, bcrypt 패키지 의존성 추가
```

**예시 2: 기타 변경사항이 없는 경우**

브랜치: `chore/define-claude-skills`

커밋 내역:
- chore: Claude 스킬 정의 파일 추가
- chore: PR 내용 작성 스킬 추가

PR 내용:

```markdown
## 주요 변경사항

> 핵심 작업 내용 요약
>
- Claude Code 스킬 정의 파일 추가 (`.claude/commands/`)
- 브랜치 생성, 커밋 메시지, PR 작성 스킬 구현
```

### 4단계: 결과 출력

분석이 완료되면:

1. PR 내용을 마크다운 코드 블록으로 출력
2. 사용자가 복사하여 사용할 수 있도록 제공

### 주의사항

- 주요 변경사항이 없으면 기타 변경사항만 작성
- **기타 변경사항이 없으면 해당 섹션을 출력하지 않음**
- 커밋 메시지와 실제 코드 변경 내용을 모두 참고하여 작성
- 민감한 정보(API 키, 비밀번호 등)는 PR 내용에 포함하지 않음
