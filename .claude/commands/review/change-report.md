# 변경 사항 보고서 스킬

현재 브랜치의 변경 사항을 분석하여 상세한 작업 보고서를 생성합니다.

## 실행 지침

### 1단계: 변경 정보 수집

```bash
# 브랜치 정보
git branch --show-current

# 분기 지점 찾기 (이 브랜치에서 커밋한 것만)
MERGE_BASE=$(git merge-base dev HEAD)

# 커밋 내역
git log $MERGE_BASE..HEAD --oneline

# 변경된 파일 통계
git diff $MERGE_BASE..HEAD --stat

# 변경된 파일 상세 목록
git diff $MERGE_BASE..HEAD --name-status

# 전체 변경 내용
git diff $MERGE_BASE..HEAD
```

### 2단계: 변경 사항 분류

변경 내용을 다음 카테고리로 분류하세요:

| 카테고리 | 설명 |
|----------|------|
| 🆕 새 기능 | 새로 추가된 기능이나 페이지 |
| 🔧 수정/개선 | 기존 기능의 수정이나 개선 |
| 🐛 버그 수정 | 버그나 오류 수정 |
| 🎨 UI/스타일 | 디자인, 스타일 변경 |
| ♻️ 리팩토링 | 코드 구조 개선 (기능 변경 없음) |
| 📦 의존성 | 패키지 추가/삭제/업데이트 |
| ⚙️ 설정 | 설정 파일 변경 |
| 🗑️ 삭제 | 제거된 코드나 파일 |

### 3단계: 보고서 작성

다음 형식으로 보고서를 작성하세요:

```markdown
## 변경 사항 보고서

### 📋 개요

- **브랜치**: `feature/xxx`
- **커밋 수**: N개
- **변경된 파일**: N개
- **추가된 라인**: +N
- **삭제된 라인**: -N

### 📝 작업 요약

> 이 브랜치에서 수행된 작업에 대한 1-2문장 요약

### 🆕 새 기능

- **[기능명]**: 기능 설명
  - 관련 파일: `src/xxx.tsx`, `src/yyy.ts`
  - 상세: 구체적인 구현 내용

### 🔧 수정/개선

- **[항목]**: 수정 내용
  - 관련 파일: `src/xxx.tsx`

### 🐛 버그 수정

- **[버그 내용]**: 수정 방법
  - 관련 파일: `src/xxx.tsx`

### 🎨 UI/스타일 변경

- 변경 내용

### ♻️ 리팩토링

- 리팩토링 내용

### 📦 의존성 변경

| 패키지 | 변경 유형 | 버전 |
|--------|----------|------|
| xxx | 추가 | ^1.0.0 |

### 📁 변경된 파일 목록

<details>
<summary>전체 파일 목록 (N개)</summary>

| 상태 | 파일 경로 |
|------|----------|
| M | src/xxx.tsx |
| A | src/yyy.ts |
| D | src/zzz.ts |

</details>
```

### 출력 규칙

1. **한글로 작성**
2. 해당 카테고리에 변경 사항이 없으면 해당 섹션 생략
3. 변경 내용을 구체적으로 설명 (단순 파일명 나열 X)
4. 중요한 변경 사항은 상세 설명 포함
5. 파일 목록이 10개 이상이면 `<details>` 태그로 접기
